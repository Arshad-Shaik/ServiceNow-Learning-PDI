# Day 03 - Users, Groups and Roles

## User

A User is an individual who can access ServiceNow.

Examples:

* Employee
* Manager
* Developer
* Administrator

## Group

A Group is a collection of users who share responsibilities.

Examples:

* Service Desk
* Application Development
* Database
* Change Management

## Assignment Group

The team responsible for handling a task or incident.

Example:
VPN issue → Network Team

## Role

A Role is a collection of permissions.

Examples:

* admin
* itil
* developer

## Difference Between User, Group and Role

User = Person

Group = Team

Role = Permissions

## Knowledge Base

A collection of articles and solutions that help users solve issues.

Example:
VPN configuration articles.

## Interview Questions

### What is a User?

An individual who can access and interact with ServiceNow.

### What is a Group?

A collection of users working together.

### What is a Role?

A collection of permissions assigned to users.

### What is an Assignment Group?

The team responsible for handling a record.
