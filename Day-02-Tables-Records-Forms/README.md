# Day 02 - Tables, Records and Forms

## Table

A table stores records in rows and columns.

**Example:**
Incident Table

## Record

A record is a single row inside a table.

**Example:**
INC0010001

## Field

A field stores a single piece of information.

**Examples:**

* Number
* Caller
* State
* Priority
* Urgency

## Form

A form is used to view and edit a single record.

**Example:**
Opening INC0010001 displays its form.

## Table vs Record

Table = Collection of records

Record = Single row of data

## Data Types

### String

Stores text.

**Example:**
Short Description

### Choice

Stores predefined values.

**Example:**
State

### Reference

Stores a relationship to another record.

**Example:**
Caller

### Date/Time

Stores date and time values.

**Example:**
Opened

## Audit History

Audit History tracks field changes made to a record.

**Example:**
State changed from New to In Progress.

## Interview Questions

### What is a Table?

A database structure that stores records in rows and columns.

### What is a Record?

A single row of data inside a table.

### What is a Field?

A single data element within a record.

### What is a Form?

A page used to view and edit a record.
